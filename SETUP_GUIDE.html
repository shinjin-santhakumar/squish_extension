Squish Helper — VS Code Extension

Setup & Testing Guide  ·  To save as PDF: open in browser → Ctrl+P → Save as PDF

What This Extension Does

Provides autocomplete inside VS Code for Python Squish test scripts. It replicates the behaviour of the Squish bat file that injects global scripts into the IDE, and bundles stubs for the Squish built-in API (test.*, object.*, waitForObject, etc.). It does not run or debug Squish tests.

Prerequisites

  • VS Code — any recent version
  • Node.js — verify by running node --version in a terminal. If missing, download from nodejs.org.
  • The extension source — cloned or pulled from your GitHub repo.

First-Time Setup (after clone or pull)

1
Open the folder in VS Code
File → Open Folder and select the Squish Extension folder. Do not open a subfolder or an individual file — VS Code needs the root.
2
Install dependencies
Open the integrated terminal (Ctrl+`) and run: 
npm install
This downloads the packages listed in package.json into node_modules/. You only need to do this once per clone, or after package.json changes.
Note The compiled out/ folder is committed to the repo, so you do not need to run npm run compile manually after a pull — it is already up to date. The F5 launch task compiles automatically before every debug session anyway.

Launching the Extension

1
Press F5 in VS Code (with the Squish Extension folder open).
VS Code will compile the TypeScript and open a second window titled [Extension Development Host]. All testing happens in that second window.
If a dropdown appears asking you to select a debugger You are missing the .vscode/launch.json file. Make sure you pulled the latest version of the repo — it should contain .vscode/launch.json and .vscode/tasks.json.
If you see "preLaunchTask npm: compile terminated with exit code 1" Run npm install in the terminal first, then press F5 again.

Configuration

In the Extension Development Host window, open Settings (Ctrl+,) and search squishHelper. Choose one of the three options below.

Option A — Eclipse project directory Recommended

Set Squish Helper: Squish Project Dir to the folder that contains your .project and .pydevproject files (the Squish Global Scripts Eclipse project). The extension reads the linked resources and resolves the actual Python source paths automatically.

C:\repos\SquishGlobalScripts

Option B — Bat file

Set Squish Helper: Bat File Path to your startsquish.bat:

C:\repos\myproject\startsquish.bat

Also set Squish Helper: Global Script Base Path to the drive root your paths resolve against (usually C:\).

Option C — Manual paths Fallback

Set Squish Helper: Global Script Dirs to a JSON array of absolute paths:

[
  "C:\\repos\\apollo\\common\\apollo",
  "C:\\repos\\wiz_utils\\common\\wiz_utils",
  "C:\\repos\\pc_sim_gui\\common\\pc_sim_gui"
]
No configuration needed for built-in stubs Even with no configuration set, autocomplete for the Squish built-in API (waitForObject, test.*, object.*, etc.) works immediately — those stubs are bundled inside the extension.

Verifying It Works

1
Check the status bar
In the Extension Development Host, open or create any .py file. Within a few seconds the bottom-right status bar should show something like: 
Squish: 120 symbols
2
Test built-in API completions
In the .py file, type each of the following and press Ctrl+Space:
What you typeExpected result
waitForObjectAppears as a Function with docstring
snoozeAppears as a Function
test.Filtered list: compare, verify, fail, warning, log…
object.Filtered list: exists, children, properties, findChildren…
MouseButton.LeftButton, RightButton, MiddleButton…
3
Test global script completions
Type the name of a function you know exists in one of your configured script directories. It should appear in the completion list. Selecting it shows the file path and line number in the detail panel.
4
Check the output log
View → Output → select Squish Helper from the dropdown. The language server logs every directory scanned and the total file count.
5
Force a rescan
Open the Command Palette (Ctrl+Shift+P) and run:
Squish Helper: Rescan Global Script Directories
The status bar count updates. Use this after changing config or editing script files.

Common Errors

preLaunchTask 'npm: compile' terminated with exit code 1

Cause: node_modules is missing — tsc is not available.

Fix: Run npm install in the terminal, then press F5 again.

A dropdown appeared asking to select a debugger when pressing F5

Cause: .vscode/launch.json is missing from the repo.

Fix: Pull the latest version of the repo. If the file still isn't there, check that .vscode/ is not listed in .gitignore.

Status bar stuck on "Squish: loading…"

Cause: The language server crashed on startup.

Fix: Open View → Output → Squish Helper to read the error. Most often out/server.js does not exist — run npm run compile.

Status bar shows "Squish: 0 symbols" after configuring paths

Cause: The scanner could not find any .py files at the resolved paths.

Fix: Open View → Output → Squish Helper to see which paths were scanned. Confirm the directories exist and contain .py files. If using Option A, make sure .project and .pydevproject are in the exact folder you configured (not a subfolder).

test. or object. shows no completions

Cause: The dot trigger did not fire, or you are not in a Python file.

Fix: Type the dot and immediately press Ctrl+Space. Also confirm the language mode in the bottom-right shows Python — if it says Plain Text, click it and change it to Python.

Warning: "Failed to parse project files in …"

Cause: The .project or .pydevproject file could not be read.

Fix: The path in squishHelper.squishProjectDir must point to the folder containing the files, not to the files themselves. Also confirm the files are named exactly .project and .pydevproject (dot prefix — they may be hidden in Windows Explorer; verify with dir /a in a terminal).

Warning: "Failed to parse bat file"

Cause: The bat file path is wrong or the file is not accessible.

Fix: Use an absolute path in squishHelper.batFilePath and confirm the file exists. The extension expects lines in the form set "KEY=VALUE" or SET KEY=VALUE.

Completions appear but selecting one inserts nothing

Cause: The language server and extension host went out of sync after a recompile.

Fix: Close the Extension Development Host window and press F5 again for a fresh session.

Extension works locally but breaks after pushing and pulling

Cause: node_modules is not committed (correct) but was also not installed after pulling.

Fix: After every pull, run npm install once. The out/ folder is committed, so no compile step is needed.

Day-to-Day Workflow

SituationWhat to do
Fresh clone / after pull Run npm install, then press F5
Making changes to the extension Edit source files in src/, then press F5 (auto-recompiles)
Global script files changed on disk Run Squish Helper: Rescan from the Command Palette
Config changed (paths, bat file, etc.) Saving settings triggers an automatic rescan
Pushing changes Commit everything including the out/ folder
